## WHAT THIS PROJECT IS

## THE THREE LAYERS

Wojaks are the ground truth. They have seeds (immutable identity) and state (mutable accretion). Pepos and memos are descriptive clusters, not structural containers. Axes are first-class, inter-memotic, and belong to no memo. Wojaks create axes by having coordinates on them.

Do not write code that treats memos or pepos as parents, owners, or containers. Do not write code that scopes axes to a memo. Do not write code that gives memos or pepos structural authority over wojaks.

## AXIS COORDINATE TYPES

Coordinates are typed: scalar, vector, ray, matrix, tensor, graph, lattice, manifold. These nest recursively. Minimum 31 dimensions, uncapped. Some coordinates are tensors, not lines. The system must be type-aware when computing distances.

Do not flatten coordinate types. Do not treat all coordinates as scalars. Do not cap dimensions.

## STORAGE

The .greg format (Fractal Content-Addressed Continuous Field) is append-only CBOR with a witness mesh. Seeds are identity (not hashes, not UUIDs). Connections are direction- and cycle-agnostic. The format imposes no topology.

The hot/warm/cold attention model bounds memory: hot = within N relational hops of current query, warm = within 2N hops (summary loaded), cold = everything else (seed on disk, woken on demand). Eviction by relational distance, not age.

Do not add indexes. Do not add schemas. Do not add hierarchy to the storage format. The rhizome is flat. Structure emerges from the data.

## QUERYING

Querying is diffusion, not traversal. A query enters the rhizome and propagates through connections, accumulating relevance by signal strength. Do not implement graph traversal algorithms. Implement signal propagation.
